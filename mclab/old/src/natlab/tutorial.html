Get your own copy of Natlab:


  1. Jar package : Go to http://www.sable.mcgill.ca/natlab/Natlab.jar to download the compiled Jar package

    OR

  2. Source Code : Natlab source code is freely provided under the Apache Version 2.0 license. Go to http://www.sable.mcgill.ca/natlab/Natlab_src.tar.gz to download the source code as a .tar.gz source archive for UNIX-like systems or http://www.sable.mcgill.ca/natlab/Natlab_src.zip to get a .zip source archive for Windows.



Build from the source code


Building on UNIX-like systems


  1. Make sure you have JDK6(or higher) and ANT installed on your system

  2. Unpack the .tar.gz source archive

  3. run cd ./Matlab\ Source/languages/Natlab

  4. run ant jar

  5. If everything goes well, you should get a Natlab.jar package

    Building on Windows

  1. Make sure you have JDK6(or higher) and ANT installed on your system

  2. Unpack the .zip source archive

  3. Open a Command window (Select Start-> Run, type “cmd” and press Enter)

  4. Go to the directory where you have downloaded the .zip archive

  1. run cd “Matlab Source\languages\Natlab”

  1. run ant build


Setting up an Eclipse environment


We use Eclipse as our primary development IDE and recommend it for anyone who would like to play around with the source code. You can follow the steps below to set up your Eclipse environment.


  1. Import Matlab Source into a workspace

  2. Open an Ant view

  3. Click on “Add Buildfiles”

  4. Select “build[default]”

  5. Wait for a few moments for the “bin-ant” directory to populate







Running Natlab


  1. cd to the directory containing Natlab.jar

  1. Run java -jar Natlab.jar [OPTION]... [FILE]

  2. Run java -jar Natlab.jar -help for a list of OPTIONs and their functionalities



Notes:

1. If Eclipse gives an error like “com.sun.tools.javac.Main is not on the classpath”, make sure that JAVA_HOME points to JDK and make sure you have the tools.lib under jdk/lib (You may copy it from jre/tools.lib)


2. You can also run the code in Eclipse. Open “Run Configurations”, click “Search” in “Main class” section, pick “Main - natlab”. Then go back to “Run Configurations” panel, click “Arguments” and type command in “Program arguments” section. For example, type “-x ~/test/test.m” into the section. Then, click “run” button at the right bottom corner.


3. When you use -x or -xml flag to get xml IR, if you encounter some error, please try put -m or -matlab following -x or -xml.
